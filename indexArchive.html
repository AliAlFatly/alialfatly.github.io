    Toymania - asp.net framework (4.6) MVC razor

    Voor mijn 4e project op school kreeg ik een opdracht om in een groep een webshop te bouwen. Mijn groep en ik kozen daarbij voor asp.net webshop met entityframework model verbinding en een SQL database.

    Toymania is een webshop voor het verkopen van speelgoed. Het is dan geen framework dat de mogelijkheid biedt voor elke verkoper om te verkoper, alleen de administrator/eigenaar verkoopt.

    Toymania heeft meerdere functionaliteiten, de gebruikers kunnen bijvoorbeeld:

    • Vanuit de home pagina de meest verkochte items inzien
    • In de store naar items zoeken en filteren op prijs/category en subcategory
    • Bij het kliken van een item de item detail inzien en een associatie inzien
    • Items aan hun wishlist/cart toevoegen
    • Uitchecken
    • Items vanuit hun wishlist toevoegen aan hun cart
    • Saldo opwarderen door middle van een coupon
    • Orders inzien
    • Track & track inzien
    Behalve de gebruikersaccount zijn er ook admin accounts, en die kunnen:
    • Coupon code genereren met een bij behoorde gekozen bedrag
    • Gebruikers accounts inzien, gebruikers bannen/unbannen
    • Alle items inzien en specifieke updaten/archiveren/toevoegen
    • Gearchiveerde items terug zetten in de store
    • Alle functionaliteiten van een normale gebruiker uitvoeren
    • Statistieken inzien

    Zie bovenstaande video voor visuale uitleg.

    Proef of concept recommender system (aanbevelingssysteem) - Dept

    Elke jaar wordt er een emerging technology project door bedrijven georganizeerd voor tweedejaarsstudenten. Voor het jaar 2018/2019 werd dit door onder andere Dept georganiseerd. De project van dept was het ontwikkelen van een proef of concept voor een aanbevelingssysteem (met behulp van machine learning/deeplearning). Ik koos voor dit project en werd daarbij ingeloot.

    Bij dit project ging het meer om de kennis van het systeem dan om code. Ik moest dan bij dit project aanbevelingssysteem bestuderen.

    Na het bestuderen van aanbevelingssysteem architectuur, machine learning algoritmes en de gekregen data van dept ging ik met mijn groep besluiten om user-based collaborative systeem architectuur te implementeren met een embedded neural netwerk algoritme. Ik was hoofdzakelijk verantwoordelijk voor het inlezen van de theory en het algoritme (wij kwamen bij dit keuze omdat ik al eerder ML heb gedaan bij een keuzevak, mijn groep heeft dan uiterraad alle kennis mee gekregen). Ik was met een groepgenoot verantwoordelijk voor het opzetten van het eind algoritme. Ik heb verder niets gedaan aan het bouwen van de website dat de algoritme bestuurt. de website (django website) werd door de overige groepsleden gebouwd.

    Ik heb met behulp van tensorflow de blocken samengesteld voor de embedded neural network. Ik heb dan niet zelf een embedded neural network van scrath ontwikkeld. Ik heb een framework gebruikt waarbij de structuur van elke component gedefineerd was. Ik heb dan de componenten (lagenstructuur, regulators, etc..) gekozen en samengesteld naar kennis van hoe ze met elkaar werken. Ik heb ook samen met mijn groep de training uitgevoerd totdat wij bij de juiste hyperparameters (trainings variabelen) zijn belandt.


    Github voor dit project is prive, een versie van de algoritme is op de foto te zien voor inzicht.

    Ik vond dit project zeer leerzaam, ik kreeg daarbij de mogelijkheid om veel te lezen en om mezelf qua machine learning te ontwikkelen. Ik kreeg zeer veel informatie en kennis van het implementeren van dit project.
